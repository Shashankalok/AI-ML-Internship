# 📘 Day 14: LLM Basics & Prompt Engineering

## 🚀 Project Overview

This project focuses on understanding **Large Language Models (LLMs)** and applying **prompt engineering techniques** using the Groq API. The goal is to explore how LLMs generate responses and how different prompting strategies affect output quality.

---

## 🎯 Objectives

* Understand how LLMs work (tokens, context window)
* Learn prompt engineering techniques
* Experiment with different types of prompts
* Integrate LLM API using Python
* Build reusable prompt templates
* Analyze output quality and cost

---

## 🛠️ Tools & Technologies

* Python
* Jupyter Notebook (VS Code)
* Groq API
* groq Python library

---

## 📚 Concepts Covered

### 🔹 Large Language Models (LLMs)

AI models that understand and generate human-like text.

### 🔹 Tokens & Context Window

* Tokens = small pieces of text
* Context window = memory limit of model

### 🔹 Temperature

Controls creativity of output:

* Low → Accurate
* High → Creative

### 🔹 Prompt Types

* Zero-shot
* Few-shot
* Chain-of-thought
* System vs User prompts

---

## 💻 Implementation

### ✅ API Integration

Connected Groq API and generated responses using Python.

### ✅ Prompt Engineering

Tested different prompt styles:

* Summarization
* Extraction
* Generation
* Reasoning

### ✅ Experiments Performed

* Temperature comparison
* Prompt variation testing
* Structured vs unstructured prompts

### ✅ Creative Use Cases

* Stand-up comedy generation (Hinglish)
* Indian hip-hop song generation

---

## 📊 Results & Observations

* Structured prompts produce better outputs
* Few-shot prompting improves accuracy
* Chain-of-thought improves reasoning
* Temperature controls creativity

Cost is based on tokens:

```
Cost = (Input Tokens + Output Tokens) × Price per token
```
